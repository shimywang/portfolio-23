Buffer Region: A Tool for Professional Map Users

Research, Instrumental Interaction, Empirical Study, GIS

02.2019 - 08.2019

Paris, France

INRIA, Ex(Situ

Supervisor:
Wendy Mackay

A research project conducted under the supervision of Wendy Mackay. Professional map users use Geographic Information Systems(GIS), a kind of specialized map application for making and analyzing maps.

However, GIS is hard to use and learn for early-career professionals. Although GIS provides a large quantity of complicated tools for its users, the interaction in GIS remained almost the same since it was created. In this project, I focus on creating tools for helping GIS professionals.

1 User Research

Procedure

Participants

I interviewed 13 professional GIS users using critical-object technique, most of them have used GIS for 1- 5 years, while one have 10+ years experience and another one just started using GIS for less than 6 months.

How it works

How users work with GIS differs according to the actual projects and their discpline. Workflow of GIS users can be summarized into 4 steps, but not all uers need to go through all of them or follow this sequence.

User Strategy

GIS professionals have very different tasks than normal map users, I notice they have four main strategies while completeing their tasks:

Use distance as core criteria for analysis

- Three most popular and basic analysis methods in GIS includes distance as core criteria

*Buffer region, Euclidean distance analysis, Network analysis

Preference of direct manipulation

- For directly editing color & style
- For adding new elements (e.g. icon for different POIs) onto map
- Need of customizing layout & contents of legends

Visualize data on map for understanding data

- Get feedback on data processing process
- Understand how data are geologically distributed, which is hard to know by text
- Rough analysis, find out focusing area or main problem for further anaysis

Overlap layers

- Overlap base map with analysis result for better understanding
- Overlap layers for conducting multi-criteria analysis

2 Design

Instrumental Interaction

I used instrumental interaction as my design guideline. This includes the following 3 guidlines and is aiming at building simple but powerful interaction techniques.

1 Reification

It reifies commands or objects.

2 Polymorphism

The new interaction tools can be applied to multiple objects of different types.

3 Reuse

The process of creating commands or the result of commands can be reused by users to other objects.

First design Alternative: Mask Layer

This design is inspired by the Mask concept that are widely used in graphical editing softwares like Photoshop and Figma. In these graphical editing softwares, mask is used as a special layer attached to a layer to edit its shape.

Users can apply mask to layer, which contains analysis methods applied to layers.

By clicking at mask, users can go back to adjust parameters, this support user tasks of exploration by reducing mental efforts of memorizing parameters when users explore different parameters.

Existing mask can be reused by applying to multiple objects. Most GIS users need to apply same analysis methods to multiple layers when completing their tasks. This could help with reducing repeated tasks.

Reflections

This is a very direct way for applying analysis methods to layers of data. With mask users can clearly see what analysis methods has been applied to what layer. However, this design only works for a few scenario, as some analysis methods require multiple layers as input, and mostly multiple analysis methods apply to one single layer, making the relationshop between mask and layers hard to read and understand.
Also, although mask better support user exploration, users still need to adjust parameters when applying one mask to multiple objects.

Second design Alternative: Bubble

I developed Bubble concept based on my first design and reflection. I found out that it is hard to create a object of interest that represent analysis methods, therefore I change my focus to the reuse of history actions.

Bubble stores parameters that have been applied to analysis methods.

Multiple bubles can be combined for issuing multiple analysis methods to one layer. In most cases GIS professionals need to process their raw data through multiple steps, and repeat similar steps to other layers of data when making one map.

This help reduce the work of applying multiple methods to multiple objects, as they are stored in bubbles and can be easily applied.

Reflection

With Bubble concept users are able to reuse previous analysis methods. Bubble is easy to understand, and probably would help GIS professionals in their works. However, with this design it still requires users to frequently change parameters when applying to different objects. Interaction with Bubbles still remained mainly on pop-up windows and text fields.

Final design: Buffer Region

Buffer region is originated from an analysis tool that GIS professionals use. By creating buffers, users could create a boundary and therefore know what are the elements within boundaries. However, if users want to change the diameter of buffer area, they have to redo the process of going through a long list of paramters.

Buffer region reifies the process of creating buffer area. However, users don't have to create by specifying parameters but can draw out buffer region freely on canvas.

Users can apply buffer region to elements on canvas by dragging it onto the target element.

Users can change the diameter of created buffer region, and switch the meaning of diameter between time and distance.

Buffer Region can also be applied to a layer and to a group of objects.

3 Reflections

Easy to understand and use for early-career map makers
Support multiple tasks

It can be used for buffer area analysis, Euclidean analysis and be extended to network analysis

Support user strategy of overlapping for analysis
Support for exploration

Uers do not need to redoing steps for exploring parameters

Processing time for changing large quantity of buffer area might be long
Users get less control on creating buffer area
