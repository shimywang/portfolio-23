Dodgeball: Mixed Reality & Tangible Interaction Game

Game Design, Augmented Reality, Tangible User Interface, Multiplayer

11.2018 - 02.2019

Paris, France

Course Project

Mixed Reality & Tangible User Interface

This is a course project aiming at developing a game utlizing interaction technology of Virtual Reality/Augmented Reallity and Tangible User Interface

Collaborated with Zaiqiao Ye, Qiwei Gao

1 Game Design

DodgeBall is an Augmented Reality game that allow users to manipulate actual object and interact with the game world.

Demo Video

The Game

Platform PC
Other Equipment A piece of paper with marker
Players 2 - 4
Developement Unity + Vuforia

In the main game scene players can see a football court divided into two parts, with a football goal and several kinds of obstacles on each part.

There are three kind of different obstacles exists in this game - pine tree, normal trees and bouncing spoonges, each of them have different bouncing rates and rebound damping. Players need to utilize these obstacles to fight against each other.

2 Play the game

Setup the game

To setup the game, players needs to print out a paper with marker and prepare a computer with camera. Players holds the paper with marker close to the camera. Players can start the game after marker being recognized.

Marker for Interaction

This game is based on recognition of markers, and players then play the game with marker.

Users are divided into two groups, and each group will hold one side of the paper with markers. Players manipulate the ball by rotating the paper, and the ball will roll to the side of the court in lower position.

Scoring

Players are divided into 2 teams and the court is also divided into 2 parts, both of which contains the goal for one team and the obstacles for the other team.

When the ball crash into the obstacles that is on the part of the court for the other team, the team can gain 5 point and when the ball goes into the area of the goal of the other team, the team can gain 10 points.

3 Reflections

One of the hardest part in this project is to come up with a nice idea that combines AR and Tangible User Interface, which also make these two techniques meaningful in the project.

We also had very interesting ideas that focused more on AR and ignored the tangible part, and ideas that focused more on enriching the interaction but less on AR, and found it hard to combine both of them in a single project, as AR focus more on displaying extra visual information and tangible user interface focus more on enriching the way users could interact with user interfaces, it is hard to focus both of them at the same time.

Another challenge we encounter is setting the physics in our game so that the player can play it smoothly. The Bounciness, frictions and gravity is tricky to set when we test it. So it takes some time for us to try what is the appropriate value for each physics attributes.
