LinkLine: Mobile Journey Planning Tool for Event Planners

Interaction Design, User Experience, Instrumental Interaction, Prototyping

10.2018

Paris, France

Course Project

Design of Interactive System

In this project we are focusing on finding out problems users have with Google Maps and deliver a design solution using design guildelines of Instrumental Interaction

Collaborated with Alina Nicolae, Zaiqiao Ye, Joel Gil Leon

1 Discovery: What is the problem and who is the user?

Below shows the process of how we find out focusing problem.

Problem we decided to focus on:

Event organizers finds it hard to compare multiple places and collaborate in Google Maps

Unable to compare multiple routes

When comparing route & transportations to multiple places, user has to search for route for every single POI and compare them manually.
Similarily, when searching route starting from different places, user has to create seperate search for each starting position.

Switching to other tools when collaborating

Organizing events often includes multiple people in making decision.
In order to share information collected from Google Maps, users often makes screenshots and discuss elsewhere.
However, when discussing about new or other alternatives, it is hard to make changes with screenshots.

2 Invention: What is possible?

Inspiration from paper maps

Let’s go back to how user interact with paper maps:

When planning for routes with multiple options, users draw sketches of possible routes on top of paper maps.

Interaction Techniques

We are inspired by the following interaction techniques:

Drag-and-Drop

Drag-and-Drop is a popular direct manipulation techniques to move an object to another context.

Marking Menu

Marking menu is a Pie Menu where you select the items with straight stroke gestures.

With marking menu, users can achieve selection in shorter time than with drop-down menus.

3 Design: What should it be?

Design process

Here is our design process:

Helping with comparing locations

When comparing routes to different locations, there are mainly two scenarios.

LinkLine as interaction tool

We created LinkLine as a tool for users to interact with POIs on map.

LinkLine is a special kind of line that represents the actual routes between locations. It is an editable, movable and collaborative tool to link locations on maps.

Users interact with LinkLine by drag-and-drop or with marking menu.

1. Tap to create LinkLine

To create a Linkline, users tap at the start location and then tap the destination.

2. Interact with LinkLine by drag-and-drop

When changing destination or starting point, users could simply drag the starting point or the destination to the new destination.

3. Interact with LinkLine by Marking Menu

LinkLine has its own attributes, including transportation methods used for finding the routes and measuring transprotation time as well as color for representing LinkLine.

Users can change transportation methods to replan the route or to change the color of LinkLine to better seperate different routes when analyzing.

Users can open marking menu by tapping at LinkLine.

Grouping POIs

Users can group multiple POIs and apply LinkLines to a group of POIs, or change attributes of LinkLines to a group of POIs to avoid repeating tasks of creating and editing LinkLines for every single POI.
Group can also be eddited, adding POIs or remove POIs from the group.

4 Reflection on Design

Real User Scenario & Real User problem

The problem LinkLine is trying to solve comes from real user scenario, coming from our interviews with event planners.

Explores One-to-Many relationships of searching for routes

Many map applications focus on routes with 1-to-1 relationships between 2 POIs, and LinkLine is an interaction tool that tries to support user tasks of acquiring 1-to-multiple POIs search without going back and forwards for searching and manually memorizing infos.

Possibility to expand to other user groups

We think LinkLine can not only be used by event planners, but other user groups, such as tourists who are planning for their routes between multiple tourist attractives.

Experience of searching for POIs

We were mainly focusing on the tasks of comparing POIs, and therefore focusing less on how users searched out POIs before they compare routes. There might contraditciton in the interface if we also include searching POIs in our prototype and will need further adjustment to make LinkLines be able to be used in real Map APP.

Collaboration & Sharing

Although we included the problem of Google Maps unable to support collaboration and complicated route sharing (on mobile APP), we didn’t focus that much on providing solution for solving this problem, there’s much more can be done in collaborating on route planning.
